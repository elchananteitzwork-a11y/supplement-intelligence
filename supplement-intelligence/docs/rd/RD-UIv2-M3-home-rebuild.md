- **Portfolio aggregates (pulse row)**: `components/dashboard/aggregates.ts` — `computeV2BuildRate`, `computeAvgQuality`, `computeLifecycleCoverage`, `computeAvgConfidence` reused verbatim, unchanged inputs/outputs (each already returns honest `null`→"—" on zero real data). `computeAvgConfidence`'s numeric percentage is what the pulse row displays as words+dots (see §3, Risks — this is a display-layer bucketing, not a new computation).

## 3. Files to change

- **`app/dashboard/page.tsx`** — merge in the `watchlist` fetch and `derivePipelineViewModel` call; replace the 7-tile `StatTile` row + `DashboardOpportunityCard` grid with the mockup's anchor sentence + 4-figure pulse row + "Needs your attention" section + stage-grouped `CandidateRow`/`StageGroup` rows. Auth/fetch skeleton otherwise unchanged.
- **New: `components/pi/HomeShell.tsx`** — the mockup's cream sticky top-nav (brand+`RotorMark`, Home/Compare/History/Track Record/Settings, quota display, CTA relabeled **"Log a hunch"** per frozen language, fixing `SideNav.tsx`'s non-compliant "+ New Analysis" *for this screen only*). **Scope decision:** this milestone does **not** touch `components/shell/SideNav.tsx` or `AppShell.tsx` in place — the audit flagged those as shared, load-bearing infrastructure across 8 routes (Compare, History, Track Record, Settings, Analyze, Memo, Watchlist, Alerts). A new, separate component is built and wired only into `/dashboard`; the other 7 routes keep the current sidebar unchanged until their own milestone in the `DESIGN_LANGUAGE_PLAN.md` §5 rollout order. This is the smallest-blast-radius way to ship Home now without a cross-route visual regression.
- **New: `components/pi/AttentionCard.tsx`** — renders the same derived attention items `derive.ts` already produces (`analysis-complete`/`stale-watch`), in the mockup's card layout. `components/pi/WhatChangedStrip.tsx` (the existing renderer for the same data) is not reused as-is since its visual doesn't match the mockup's card spec — it becomes unused by this page (left in place; not deleted, see Non-Goals).
- **`components/ui/StatTile.tsx`** — no change; consumed as-is for 3 of 4 pulse figures.
- **New: small confidence-bucketing helper** (e.g. `components/pi/confidenceTier.ts`) — pure function `(pct: number) => { label: 'Limited'|'Moderate'|'High', dotsFilled: 1|2|3 }`, thirds split (`<34` / `34–66` / `≥67`). Takes the *already-computed* `computeAvgConfidence` percentage as input — no new scoring, purely a display bucket. **Flagged explicitly (see Risks): these thresholds and labels are a presentation convention, not sourced from an existing tiering system** — the codebase's one existing HIGH/MODERATE/LOW confidence label (`components/memo/InvestmentThesis.tsx:34`, `CONFIDENCE_LABEL`) turns out to feed a *different* real number (`computeVerdictConfidence(memo.expandable_cards)`, a source-agreement count) than the one the pulse row displays (`computeConfidenceAssessment().overallConfidence`, the weakest-link float) — reusing that label set would silently mix two unrelated confidence concepts, which is exactly the kind of "two different confidence numbers for the same analysis" bug `derive.ts`'s own comments say was already fixed once (§3 of the architecture audit). New three-word vocabulary (Limited/Moderate/High) avoids colliding with the existing HIGH/MODERATE/LOW label set that means something else.
- **Delete: `app/pipeline/page.tsx`, `app/pipeline/preview/page.tsx`.** Confirmed zero inbound `href`/`redirect` references anywhere in the repo (grepped) — this is a dead-code removal, not a live cutover. Remove `'/pipeline'` from `middleware.ts:9`'s `GUARDED` array in the same change.
- **No change**: `components/dashboard/DashboardOpportunityCard.tsx`, `components/dashboard/aggregates.ts` (aggregates reused, the card component becomes unused by this page — left in place, see Non-Goals), `components/ui/VerdictBadge.tsx` (legacy, untouched), `lib/scoring.ts`, `lib/confidence/*` (read-only).

## 4. Risks

- **Confidence bucketing thresholds are invented, not sourced** (see §3) — thirds-split is a reasonable default, not derived from product research. Flagging for owner review post-ship rather than blocking on it now, since it's a low-stakes display convention (adjusting a threshold later is a one-line change, not a re-architecture).
- **Cross-route shell divergence, deliberately accepted this milestone**: `/dashboard` gets a top-nav; the other 7 `AppShell`-routes keep the sidebar. A user navigating Home → Compare will see the nav chrome change shape mid-session until Compare's own milestone ships. This is the explicit, smallest-scope tradeoff (see Files to Change) — named here so it isn't mistaken for an oversight when QA notices it.
- **`derive.ts`'s "evidence age" is really "time since analysis creation," not "time since shortlisted or last re-checked"** (audit finding) — `watchlist.created_at`/`last_checked_at` exist as real columns but aren't read today. Carried over unchanged from `/pipeline`; not fixed in this milestone (would be new derivation logic, out of a presentation-only scope) but named so the "X days old" copy in Attention cards isn't read as more precise than it is.
- **`Sources →` destination on the pulse row is unresolved** (mockup's own footer note) — implemented as a dead link (`href="#"`) or simplest honest placeholder until a real target is decided; not blocking this milestone since Home's per-row `→` already opens real detail (the memo page), which is where the bulk of "sources" already lives today.
- **Dead code, not deleted**: `DashboardOpportunityCard.tsx` and `WhatChangedStrip.tsx` become unused by `/dashboard` but aren't confirmed unused *everywhere* — left in place rather than guessed at (see Non-Goals).
- **`/pipeline` deletion has no staged-cutover risk in the traditional sense** — confirmed zero inbound links/redirects; it was already "reachable by URL only, not linked from nav" per its own header comment. The real user-facing change is `/dashboard`'s content changing for every authenticated user on next visit — that's what QA's live-browser pass must cover, not the `/pipeline` deletion itself.
